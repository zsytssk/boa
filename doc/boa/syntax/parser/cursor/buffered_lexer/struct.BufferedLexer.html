pub(super) fn peek(
    &mut self,
    skip_n: usize,
    skip_line_terminators: bool
) -> Result<Option<&Token>, ParseError>[src]

Peeks the nth token after the next token.

Note: n must be in the range [0, 3]. i.e. if there are tokens A, B, C, D, E and peek(0, false) returns A then:

  • peek(1, false) == peek(1, true) == B.
  • peek(2, false) will return C. where A, B, C, D and E are tokens but not line terminators.

If skip_line_terminators is true then line terminators will be discarded. i.e. If there are tokens A, \n, B and peek(0, false) is A then the following will hold:

  • peek(0, true) == A
  • peek(0, false) == A
  • peek(1, true) == B
  • peek(1, false) == \n
  • peek(2, true) == None (End of stream)
  • peek(2, false) == B
